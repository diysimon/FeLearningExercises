# Task 04

体验到代码帮你自动化完成工作的乐趣吗？

而且刚才那个任务已经有那么一丢丢“前端工程化”的微小雏形。

## 任务说明：

大概率，你刚才的脚本，是把 json 文件中的数据遍历后依次生成 HTML。

你可以在脚本中进行日志打点，然后测测每次完整生成一次的耗时是多少，可能现在数据量没那么大，耗时不多，但如果假设，你要处理的数据内容是百万级、千万级的时候，我们就不能每次生成时，都做全量的生成。

接下来，我们来实现一个增量生成的逻辑。

每次脚本运行时，只针对有变更的数据，生成对应的 HTML。
但注意，这里不需要做什么内容管理系统，或是要手动去标识、输入哪个数据发生变化。
你需要验证的流程就是：
- 第一次是全量输出
- 直接修改 mock 数据中的某一条
- 再次运行脚本，只会更新输出变更数据对应的 HTML
- 测一下耗时是否相对全量更新的变化

## 练习建议

- 尽可能不使用任何三方的库和框架，相关三方库、框架的练习会在后面有布置安排。
- 完全按照功能描述实现，不用额外做东西。
- 根据你实际情况，给自己限定一个完成时间。
