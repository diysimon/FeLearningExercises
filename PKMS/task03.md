## 任务说明：

参考 asset/mock_data_02.json，你现在多了很多数据，把这些数据也变成知识库的各个 HTML 文件吧。

当然，我现在不会再要求你用复制、粘贴的方式。也许你听说过 SPA、RIA 这样的概念，很好，不过我们暂时先不走到那一步。

因为我们目前的知识库还是以内容浏览为主，并不是一个编辑创造的复杂应用。纯浏览而言，我们希望它每个页面还是尽可能简单。

现在你需要做的事情，是写一个脚本，把你刚才复制、粘贴以及其它可能做的事情，变成一个自动化的过程。

你需要搭建一个 Node.js 环境，然后写一个脚本，输入是 mock_data_02.json，输出是一个个独立的 HTML。

另外我要求，这个 HTML 中，不要有“打开页面后进行 Markdown 数据解析的过程”，也就是说，这个解析过程是在你的脚本产生 HTML 文件的过程中就完成了，HTML 的内容部分就是干干净净的纯 HTML。
