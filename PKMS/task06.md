## 任务说明：

我们重新回到任务1的代码。在任务1里，我们所有的内容都是通过读取 json 文件数据，然后通过点击不同目录时，获取目录对应内容的索引，找到具体内容，完成渲染。

但我们也发现，任务1的代码，有个小问题，就是每次刷新页面，都会回到默认的内容展现，因此我们在任务2-5中，把内容拆到了不同的 HTML 中，来保证浏览体验。

但在任务 6 中，我们开始学习现代 SPA 中的一个小基础能力，如何在单页面中，实现 Hash 模式路由。

你需要先了解 URL 地址中，#，？分别的含义，以及在页面访问时，#，？之后的内容发生变化时，浏览器的对应行为是什么。

然后，你需要具体实现的功能为：

基于任务1，实现 task06.html，点击任何一个文章目录节点时，进行对应的路由，页面本身不发生跳转，只在目录激活节点和右侧内容区域进行局部变化。

其它要求：

- 把前面几个任务都代码挪到一个单独的目录下
- 使用 typescript 重新进行编码
- 文件名统一使用有语义的英文小写，可以使用下划线进行分隔（例如：list_table.ts, error_report.css）
