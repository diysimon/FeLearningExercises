# Task 07

我们现在开始重新审视一下我们的知识库，之前我们的知识库都只有读取展现的能力，接下来，我们给它加上新建文章的能力。

## 任务说明：

我们继承 task06 中完成的代码，在其基础上增加新功能。

在页面左侧目录树的最上方，增加一个添加内容的按钮。

点击按钮后，右侧内容变成编辑区域。

编辑区域包括三部分内容：

- 文章标题输入框（Input），当用户点击按钮，出现右侧编辑区域时，键盘输入焦点会自动进入标题输入框中，输入框默认 placeholder 文案为“输入文章标题（1-40字）”。
- 文章内容多行输入框（Textarea），placeholder 文案为“输入正文内容（不超过1000字）”
- 文章保存和取消按钮

当用户点击取消按钮时，如果文章标题和内容均不为空的情况下，会弹出二次确认弹窗，确认是否取消当前输入。用户确认取消后，则返回知识库首页（可以是第一个内容页）

当用户离开页面（关闭，刷新，前进后退等）时，如果文章标题和内容均不为空的情况下，会弹出二次确认弹窗，确认是否离开当前页面。

用户点击保存按钮时，判断标题输入框输入是否符合1-40字范围的要求，以及正文内容是否超过5000字，如果符合，则进行文章内容保存，如果不符合，则弹出 alert，告诉用户问题是什么。

文章顺利保存后，新添加的文章会出现在左侧目录中，右侧则展示刚刚添加的文章内容。

在本练习中，文章所有数据均保存在浏览器本地存储中（学习了解 LocalStorage, indexedDB 等知识，本练习建议先使用 LocalStorage 作为本地存储方案）

注意，初始时，数据为空，本练习不需要 mock 数据。task06 中读取数据的逻辑需要进行修改，不再从 mock 数据中读取，而是从 LocalStorage 中读取数据。
