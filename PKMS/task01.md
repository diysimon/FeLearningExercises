要不，这样吧，你自己做一个如何？

## 任务说明：

我们先实现一个超级超级超级简易的个人知识库。

我为你提供了现成的 mock 数据可控参考，数据在 asset/mock_data_01.json

功能需求如下：

### 整体布局

- 页面顶部，有一个大约几十像素高度的横条顶 bar，顶 bar 左侧是一个页面标题（Logo），你可以给你的个人知识库起个有意思的名字，然后把文字作为 Logo。
- 顶 bar 下面是知识管理区域，包含目录和内容两个区域
- 目录和内容两个区域为左右布局，左侧是固定宽度的知识目录，右侧是具体的知识内容区域，宽度根据浏览器窗口的宽度进行自适应适配

### 功能描述

左侧目录为一个上下排列的内容节点列表，数据来源于一个脚本文件中。(在 asset 中提供了一个可直接使用的数据参考文件 mock_data_01.json)

在目录中，会展示标题，鼠标滑动过（hover）目录中某个节点时，这个节点会展现一个稍微不太一样的小样式，同时会有个小浮层紧紧贴在鼠标附近，展示这个内容创建的时间，格式使用“年-月-日”。

点击目录中某一个内容节点后，页面在不进行跳转的情况下，会在右侧展现对应的内容。同时，点击的那个节点也会展现出一个特殊的样式，标明这个当前浏览的是这个节点。

在右侧内容区域，会分成上下两部分，上面是一个几十像素高的一个区域，显示当前浏览的内容的标题（我们通常称之为面包屑）。

下面区域就是正常的内容展示区域，也是从 day01_mock.json 中读取的，内容采用的是 Markdown 格式，你需要将其转变为 HTML 进行正确的展现。

当然，如果你现在搜索了 Markdown 是什么，或者你之前就知道它，相信你大概也在想，是我第一天自己写一个 Markdown 解析吗？当然不完全是，全部实现工作量很大，我们在模拟数据里，只用到几种 Markdown 的描述标记，你只需要先实现数据中使用的那几种即可。

上面所有的样式细节，你自行发挥，没有要求。

## 练习建议

- 尽可能不使用任何三方的库和框架，相关三方库、框架的练习会在后面有布置安排。
- 完全按照功能描述实现，不用额外做东西。
- 根据你实际情况，给自己限定一个完成时间。
